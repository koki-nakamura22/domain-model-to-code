# ADR-001: ドメインモデル図のフォーマットにMermaidを採用する

## ステータス
Accepted

## コンテキスト
ドメインモデル図を作成し、そこからコードを生成するワークフローを構築したい。図のフォーマットとして主に以下の候補があった：

- **Mermaid** — テキストベースの図記述言語
- **draw.io** — GUIベースの作図ツール（.drawio XML形式）

## 決定
**Mermaidをドメインモデル図のSingle Source of Truthとして採用する。**

draw.ioはプレビュー・説明用途で必要に応じて生成するが、原本としては使用しない。

## 理由

### Mermaidの利点
- **Git管理との相性**: テキストベースなので差分が人間に読める。PRレビューで図の変更を確認できる
- **コード生成との相性**: テキストパースが容易。LLMが読み書きしやすく、パーサーによる機械的な変換も確実
- **表現力**: クラス図として集約・エンティティ・値オブジェクトを表現するのに十分。`namespace`やコメントで集約境界も表現可能
- **プレビュー**: VSCode拡張でリアルタイムプレビュー可能。GitHubでも自動レンダリングされる

### draw.ioを原本にしない理由
- XMLは本質的にビジュアル情報（座標、スタイル）がメインでセマンティクスが弱い
- 関連線の多重度がXML属性のどこに入るか不定
- 手動で配置を変えるとXML構造自体が変わることがある
- draw.io → Mermaid、draw.io → コード、いずれの変換も脆くなる

## 結果
- ドメインモデルの編集はMermaidファイルに対して行う
- draw.ioが必要な場面ではMermaidから変換して生成する（逆方向の同期は行わない）
- コード生成のインプットはMermaidファイルとする
