## 2. ビジネスルール

### 2.1 予約ルール 🟢

**ステータス遷移:**
```
仮予約 → 確定 → チェックイン済 → チェックアウト済
  ↓        ↓
 失効    キャンセル済
           確定 → ノーショー
```

**予約の作成（仮予約）:**
- 予約には「宿泊日（チェックイン日〜チェックアウト日）」「部屋タイプ」「人数構成（大人/子供）」「料金プラン」が必須
- 空室がある場合のみ予約可能（同じ部屋タイプの在庫数を超えない）
- 仮予約の時点で在庫を一時的に確保する
- 仮予約のTTLは15分。超過で自動失効し、在庫を戻す
- 同一ゲストが同一期間に複数予約を持つことは許可する（家族分の複数部屋等）

**仮予約 → 確定:**
- 決済完了をもって確定に遷移する
- 確定時に予約番号を発行する

**予約の変更:**
- 日程・部屋タイプ・人数・プランの変更が可能
- 変更時点で空室がなければ変更不可
- チェックイン日以降は変更不可（当日変更はフロント対応＝スコープ外）

**予約のキャンセル:**
- チェックイン前であればキャンセル可能
- キャンセルポリシーに基づいてキャンセル料が発生
- チェックイン後のキャンセルは不可

### 2.2 料金ルール 🟢

**料金計算式:**
```
最終料金 = Σ(各泊ごとの料金)  − 連泊割引

各泊の料金 = 基本料金(部屋タイプ)
            × 季節・曜日係数
            + 人数調整額
            + プランオプション料金
```

**季節・曜日による変動:**
- シーズン区分（オフ/レギュラー/ハイ/ピーク）をホテルごとに期間定義
- 曜日区分（平日/金曜/土曜・祝前日）
- シーズン × 曜日の組み合わせで料金係数が決まる（例: ハイシーズン×土曜 = 1.5倍）

**人数構成による変動:**
- 部屋タイプごとに「定員」と「基準人数」を持つ
- 基準人数未満 → 減額
- 基準人数超過 → 追加料金（エキストラベッド等）
- 子供の年齢区分: 幼児(0-5歳) / 小学生(6-11歳) / 大人(12歳〜)
- 幼児: 添い寝無料、小学生: 大人料金の50%

**料金プラン:**
- 素泊まり（基本）
- 朝食付き（+固定額/人）
- 2食付き（+固定額/人）
- その他ホテル独自プラン（拡張可能な設計にする）

**連泊割引:**
- 3泊以上: 5%OFF
- 7泊以上: 10%OFF
- ホテルごとに割引率を定義可能

**キャンセルポリシー:**
- ホテルごとにポリシーを定義
- 例: 7日前まで無料 / 3〜6日前 30% / 前日 50% / 当日 100%
- キャンセル料は最終料金に対するパーセンテージ

### 2.3 チェックイン / チェックアウトルール 🟢

**チェックイン:**
- 予約が「確定」ステータスの場合のみチェックイン可能
- チェックイン可能時刻: 15:00〜（ホテルごとに設定可能）
- チェックイン日より前のチェックインは不可
- チェックイン時に実際の部屋番号を割り当てる（部屋タイプ→具体的な部屋）

**チェックアウト:**
- チェックアウト時刻: 〜10:00（ホテルごとに設定可能）
- チェックアウト完了で部屋が空室に戻る
- レイトチェックアウトはスコープ外（拡張領域として将来対応可能）

**ノーショー:**
- チェックイン日の翌日までにチェックインがなかった場合、ノーショー扱い
- ノーショーの場合、宿泊料金の100%を請求

## 3. ドメインイベント候補

| # | イベント名 | トリガー | 後続処理の例 |
|---|-----------|---------|-------------|
| DE-1 | 仮予約作成 (ReservationHeld) | ゲストが予約を申し込み、空室確認が通った | 在庫の一時確保、TTLタイマー開始、決済画面への遷移 |
| DE-2 | 仮予約失効 (ReservationExpired) | TTL(15分)を超過した | 在庫の解放 |
| DE-3 | 予約確定 (ReservationConfirmed) | 決済完了 | 予約番号発行、確認メール送信、在庫の確定反映 |
| DE-4 | 予約変更 (ReservationModified) | ゲストが日程・部屋タイプ等を変更 | 料金再計算、差額決済or返金、変更通知送信 |
| DE-5 | 予約キャンセル (ReservationCancelled) | ゲストがキャンセルを申請 | キャンセル料計算・請求、在庫の解放、キャンセル通知送信 |
| DE-6 | チェックイン (GuestCheckedIn) | フロントでチェックイン処理 | 部屋番号割当、部屋ステータスを「使用中」に変更 |
| DE-7 | チェックアウト (GuestCheckedOut) | フロントでチェックアウト処理 | 部屋ステータスを「清掃待ち」に変更、最終精算 |
| DE-8 | ノーショー検出 (NoShowDetected) | チェックイン日翌日にチェックインがない | 全額請求、在庫の解放 |
| DE-9 | 決済完了 (PaymentCompleted) | 決済システムから成功応答 | 予約確定への遷移トリガー |
| DE-10 | 決済失敗 (PaymentFailed) | 決済システムから失敗応答 | 仮予約の失効、ゲストへの通知 |
| DE-11 | 部屋ステータス変更 (RoomStatusChanged) | 管理者/清掃スタッフがステータスを変更 | 在庫反映（清掃完了→空室で空室数増加） |

## 4. 用語集

| 用語 | 英語名 | 説明 |
|------|--------|------|
| ホテル | Hotel | チェーン内の1つの宿泊施設。部屋・料金・ポリシーを持つ |
| 部屋タイプ | RoomType | シングル、ツイン、スイート等の部屋の種別。定員・基準人数・基本料金を持つ |
| 部屋 | Room | 物理的な1部屋。部屋番号を持ち、特定の部屋タイプに属する |
| ゲスト | Guest | 宿泊するお客様。予約の主体 |
| 予約 | Reservation | ゲストが特定の期間・部屋タイプで宿泊を申し込んだもの |
| 仮予約 | Held Reservation | 決済前の一時的な予約。在庫を15分間確保する |
| 予約番号 | Reservation Number | 予約確定時に発行される一意の識別番号 |
| チェックイン日 | Check-in Date | 宿泊開始日 |
| チェックアウト日 | Check-out Date | 宿泊終了日（この日の朝に退室） |
| 宿泊日 | Stay Date | チェックイン日からチェックアウト日の前日まで。料金計算の単位 |
| シーズン区分 | Season | オフ/レギュラー/ハイ/ピーク。ホテルごとに期間を定義 |
| 曜日区分 | Day Type | 平日/金曜/土曜・祝前日。料金係数に影響 |
| 料金係数 | Rate Multiplier | シーズン×曜日で決まる基本料金への掛け率 |
| 基準人数 | Standard Occupancy | 部屋タイプの料金設定の基準となる人数 |
| 定員 | Max Occupancy | 部屋タイプに宿泊できる最大人数 |
| 人数調整額 | Occupancy Adjustment | 基準人数との差分による加算/減算額 |
| 料金プラン | Rate Plan | 素泊まり、朝食付き等の宿泊プラン |
| 連泊割引 | Length-of-Stay Discount | 連泊数に応じた割引（3泊5%OFF、7泊10%OFF等） |
| キャンセルポリシー | Cancellation Policy | キャンセル料の発生条件を定めたルール |
| キャンセル料 | Cancellation Fee | キャンセル時に発生する料金。ポリシーに基づきパーセンテージで算出 |
| ノーショー | No-show | チェックイン日に現れなかったゲスト。全額請求 |
| 決済 | Payment | 宿泊料金の支払い処理。本プロジェクトではモック実装 |
